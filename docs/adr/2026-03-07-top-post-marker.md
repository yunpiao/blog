# ADR: 文章置顶标记实现

- 日期：2026-03-07
- 状态：Accepted

## 背景

首页和文章归档页已经使用 `isTop` 为文章渲染 `[置顶]` 标记，但该字段此前不影响排序，导致被标记的文章仍然按发布日期混在普通文章中。

现在需要把 `isTop` 升级为真正的置顶语义：首页和 `/post/` 列表页都必须优先展示 `isTop: true` 的文章，同时尽量不改现有模板结构、年月分组和分页 UI。

## 决策

1. 继续复用 `isTop`，不新增 `pin`、`sticky`、`featured` 等字段。
2. 首页直接采用“置顶优先 + 日期倒序”的单一有序集合：
   - 先取全部 post 并按日期倒序排列。
   - 再分成 `isTop: true` 与非置顶两组。
   - 最后按“置顶在前、普通在后”拼接为最终集合。
3. `/post/` 列表页继续在分页前使用同一套“置顶优先 + 日期倒序”排序规则，但第 1 页会把当前分页结果里的置顶文章单独渲染在顶部区块；同页归档分组只渲染剩余普通文章，避免重复展示。
4. `[置顶]` 标签的视觉标记继续保留，排序语义与展示语义共用同一个 front matter 字段。

## 后果

- 现有文章的 front matter 契约保持稳定，无需迁移内容。
- 首页前 20 条会直接按置顶优先展示。
- `/post/` 第 1 页可以把置顶文章稳定显示在页面最上方，同时保留原有年月分组和分页导航。
- `/post/` 的分页仍基于全局排序后的集合，不会因为顶部区块而在同一页重复文章。
- `isTop` 不再是纯样式字段，后续新增或调整置顶文章时会同时影响标签展示和列表顺序。
